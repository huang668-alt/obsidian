LangChain 代理基于 LangGraph 构建，因此它们支持相同的流式堆栈，并针对消息、工具调用、状态和自定义更新进行以代理为中心的投影。对于大多数应用程序和前端用例，请使用**事件流**。`stream_events(..., version="v3")`事件流返回一个带有类型化投影的运行对象，因此可以独立使用每个投影，而无需解析流模式元组。
``` Python
from langchain.agents import create_agent


def get_weather(city: str) -> str:
    """Get weather for a city."""
    return f"It's always sunny in {city}!"


agent = create_agent(
    model="gpt-5-nano",
    tools=[get_weather],
)

stream = agent.stream_events({
    "messages": [{"role": "user", "content": "What is the weather in SF?"}],
}, version="v3")

for message in stream.messages:
    for delta in message.text:
        print(delta, end="", flush=True)

final_state = stream.output
```
## 你可以观看什么

| 投影                    | 使用                        |
| --------------------- | ------------------------- |
| `for event in stream` | 包含完整包络线和所有通道访问权限的原始协议事件。  |
| `stream.messages`     | 模型消息流，每个 LLM 调用一个。        |
| `message.text`        | 消息的文本增量和最终文本。             |
| `message.reasoning`   | 推理内容暴露模型的推理增量。            |
| `message.tool_calls`  | 工具调用参数块和已完成的工具调用。         |
| `message.output`      | 模型调用完成后的最终消息对象。           |
| `stream.values`       | 代理状态快照。                   |
| `stream.output`       | 最终代理状态。                   |
| `stream.subgraphs`    | 嵌套图运行（子代理和普通子图）。          |
| `stream.extensions`   | 定制变压器投影。                  |
| `stream.tool_calls`   | 工具执行生命周期、输入、输出增量、最终输出和错误。 |

## 代理消息

`stream.messages`当您需要每次 LLM 调用输出模型结果时，请使用此功能。
``` Python
stream = agent.stream_events(input, version="v3")

for message in stream.messages:
    print(f"[{message.node}] ", end="")
    for delta in message.text:
        print(delta, end="", flush=True)

    full_message = message.output
    usage = full_message.usage_metadata
    if usage:
        print(usage)
```

`message.output`它会返回最终的 AI 消息，包括特定于提供商的内容块。在 TypeScript 中，`message.usage`当您只需要标记计数或其他使用元数据时，可以使用此方法；在 Python 中，可以从 . 读取使用情况`message.output.usage_metadata`。

## 推理内容

推理内容使用与文本内容相同的形状，但只有当选定的模型发出推理块时才可用。
``` Python
stream = agent.stream_events(input, version="v3")

for message in stream.messages:
    for delta in message.reasoning:
        print(f"[thinking] {delta}", end="", flush=True)

    for delta in message.text:
        print(delta, end="", flush=True)
```

## 工具调用

有两种有用的工具调用预测：

- `message.tool_calls`在模型生成工具调用时，流式传输工具调用参数块。
- `stream.tool_calls`从工具调用开始后，流式传输工具执行的生命周期。

``` Python
stream = agent.stream_events(input, version="v3")

for message in stream.messages:
    for chunk in message.tool_calls:
        print(f"tool call chunk: {chunk}")

    finalized = message.tool_calls.get()
    if finalized:
        print(f"finalized tool calls: {finalized}")

for call in stream.tool_calls:
    print(f"{call.tool_name}({call.input})")
    for delta in call.output_deltas:
        print(delta, end="", flush=True)
    print(call.output, call.error)
```

## 流式子代理

当一个`create_agent`调用（通常通过包装工具）触发另一个同名代理时`create_agent`，内部代理的事件流会遵循嵌套命名空间。`name=`您传递给`get_namespace` 的参数用于`create_agent`标识流中的内部代理，因此您可以按代理进行过滤和标记。
命名的子代理会显示在专用`stream.subagents`投影上。每个句柄都会暴露内部代理自身的属性`.messages`，以及（您传递的）和（调度子代理的工具调用）。由于这里只显示命名的运行，`.values`因此您无需过滤掉普通的子图。`.tool_calls``.output``.name``name=``.cause``create_agent`

``` Python
from langchain.agents import create_agent
from langchain.chat_models import init_chat_model


def get_weather(city: str) -> str:
    """Get weather for a given city."""
    return f"It's always sunny in {city}!"


weather_agent = create_agent(
    model=init_chat_model("openai:gpt-5.5"),
    tools=[get_weather],
    name="weather_agent",
)


def call_weather(query: str) -> str:
    """Query the weather agent."""
    result = weather_agent.invoke({"messages": [{"role": "user", "content": query}]})
    return result["messages"][-1].text


supervisor = create_agent(
    model=init_chat_model("openai:gpt-5.5"),
    tools=[call_weather],
    name="supervisor",
)

stream = supervisor.stream_events(
    {"messages": [{"role": "user", "content": "What's the weather in Boston?"}]},
    version="v3",
)

for subagent in stream.subagents:
    print(f"{subagent.name}: ", end="")
    for message in subagent.messages:
        for token in message.text:
            print(token, end="", flush=True)
    print()
```

`StateGraph`从工具中调用的普通子图也会出现在`stream.subgraphs`——设置`name=`以`.compile(name=...)`获取标签`subagent.graph_name`。`stream.subagents``create_agent`是指定子代理的聚焦视图，而`stream.subgraphs`涵盖所有嵌套图。请使用与您的用户界面相匹配的视图。

## 多重投影

对于异步代码中的并发消费，请`astream_events`使用`asyncio.gather`：

``` Python
import asyncio

stream = await agent.astream_events(input, version="v3")

async def consume_messages():
    async for message in stream.messages:
        print(await message.text)

async def consume_tool_calls():
    async for call in stream.tool_calls:
        print(call.tool_name, call.input)

await asyncio.gather(consume_messages(), consume_tool_calls())
```

对于同步代码，请改用`stream.interleave(...)`：
```Python
stream = agent.stream_events(input, version="v3")

for name, item in stream.interleave("messages", "tool_calls", "values"):
    if name == "messages":
        print(item.text)
    elif name == "tool_calls":
        print(item.tool_name, item.input)
    elif name == "values":
        print(item)
```

要访问未作为类型化投影公开的通道，或检查完整的事件包络，请迭代原始协议事件：

```Python
for event in stream:
    print(event["method"], event["params"]["namespace"], event["params"]["data"
```

## 自定义更新

当您的应用程序需要没有内置的投影时，例如检索进度、工件或特定领域的事件，请使用自定义流转换器。

```Python
stream = agent.stream_events(
    input,
    version="v3",
    transformers=[ToolActivityTransformer],
)

for activity in stream.extensions["tool_activity"]:
    print(activity)
```

### 在中间件上注册转换器

中间件注册转换器需要`langchain>=1.3.2`。

中间件可以在其钩子和工具旁边声明流转换器工厂。不同语言的工厂结构有所不同：`transformers`将子类的属性设置`AgentMiddleware`为工厂序列。每个工厂的形状为 `<model_name>` `Callable[[tuple[str, ...]], StreamTransformer]`，调用方式为 `<model_name>` `factory(scope)`，其中`<model_name> `scope`` 是 mini-mux 的作用域元组（`()`对于根 mux，非空值表示子图）。每次调用都返回一个新的转换器，从而保证每个子图的隔离性。

```Python
from langchain.agents import create_agent
from langchain.agents.middleware import AgentMiddleware


class ToolActivityMiddleware(AgentMiddleware):
    transformers = (ToolActivityTransformer,)


agent = create_agent(
    model="gpt-5-nano",
    tools=[get_weather],
    middleware=[ToolActivityMiddleware()],
)
```

在编译时，`create_agent`它会将中间件注册的工厂与传递给它自身`transformers=`参数的任何内容合并。编译后的图上的最终顺序为：

1. 内置的`ToolCallTransformer`。
2. 中间件注册工厂，按中间件顺序排列。
3. 来电者`transformers=`提供`create_agent`。

这样一来，内置工具调用投影就始终位于消费者变压器之前，并且调用者提供的条目拥有最终决定权。内置功能`PIIMiddleware`使用此钩子从流式线输出中删除 PII。通过此钩子`apply_to_output=True`，其注册转换器会在文本增量、工具调用参数、工具输出和状态快照离开运行之前，清除检测到的 PII，从而关闭`after_model`状态级删除功能原本会导致原始 PII 传递给实时读取器的情况`stream_events(version="v3")`。

```Python
from langchain.agents import create_agent
from langchain.agents.middleware import PIIMiddleware

agent = create_agent(
    model="gpt-5-nano",
    tools=[],
    middleware=[
        PIIMiddleware("email", strategy="redact", apply_to_output=True),
    ],
)
```
