LangChain 实现了一个流式系统来显示实时更新。对于基于 LLM 构建的应用程序而言，流式传输对于提升其响应速度至关重要。通过逐步显示输出，即使在完全响应准备就绪之前也能实现，流式传输能够显著改善用户体验 (UX)，尤其是在处理 LLM 的延迟问题时。
## 概述

LangChain 的流式系统允许您将代理运行的实时反馈呈现给您的应用程序。LangChain 流媒体技术能实现哪些功能：

-  **流式传输代理进度**——在代理的每个步骤之后获取状态更新。
-  **流式 LLM 标记**——在生成时流式传输语言模型标记。
-  **流式思维/推理标记**——表面模型推理的生成过程。
-  **流式自定义更新**——发出用户定义的信号（例如，`"Fetched 10/100 records"`）。
-  **流式传输多种模式**——可选择`updates`（代理进度）、`messages`（LLM 令牌 + 元数据）或`custom`（任意用户数据）。

## 支持的流模式

将以下一种或多种流模式作为列表传递给`stream`or`astream`方法：

| 模式         | 描述                                                          |
| ---------- | ----------------------------------------------------------- |
| `updates`  | 代理程序每执行一步操作后，都会传输状态更新。如果在同一步骤中进行多次更新（例如，运行多个节点），则这些更新会分别传输。 |
| `messages` | `(token, metadata)`从调用 LLM 的任何图节点获取元组流。                     |
| `custom`   | 使用流写入器从图节点内部流式传输自定义数据。                                      |
## 代理进度

要流式传输代理进度，请使用`stream` or `astream`方法`stream_mode="updates"`。这将在代理执行每个步骤后发出一个事件。例如，如果您有一个代理调用某个工具一次，您应该会看到以下更新：

- **LLM节点**：`AIMessage`带有工具调用请求
- **工具节点**：`ToolMessage`执行结果
- **LLM节点**：最终AI响应

传递一个`thread_id`via`config`以便对话被检查点保存，后续回合可以恢复相同的历史记录。`thread_id` 与 无关 `stream_mode`；
你可以在每次运行代理时，通过 `context` 参数传递一些自定义数据，然后在你的工具函数中通过 `runtime.context` 来读取这些数据。

## 简单理解

```python
from langchain.agents import create_agent  # 从langchain导入创建代理的函数（注：此API在较新版本中可能已变更）
from langchain_core.utils.uuid import uuid7  # 导入生成UUID v7格式的工具函数
from langgraph.checkpoint.memory import InMemorySaver  # 导入内存检查点保存器，用于状态持久化

def get_weather(city: str) -> str:
    """获取指定城市的天气信息"""
    return f"It's always sunny in {city}!"  # 模拟天气数据，总是返回晴天


agent = create_agent(  # 创建一个AI代理实例
    model="openai:gpt-5.5",  # 指定使用的模型
    tools=[get_weather],  # 注册工具列表，这里只包含天气查询函数
    checkpointer=InMemorySaver()  # 使用内存检查点保存对话状态
)

config = {"configurable": {"thread_id": str(uuid7())}}  # 配置参数，生成唯一会话线程ID

stream = agent.stream_events(  # 启动流式事件处理
    {"messages": [{"role": "user", "content": "What is the weather in SF?"}]},  # 用户输入消息
    config=config,  # 传入配置（包含线程ID）
    version="v3",  # 指定API版本（注：此参数在当前版本中可能不支持）
)

# 遍历流式事件，分别处理消息和工具调用
for kind, item in stream.interleave("messages", "tool_calls"):  # 交错获取消息和工具调用事件
    if kind == "messages":  # 如果是消息类型
        for token in item.text:  # 遍历消息文本的每个token
            print(token, end="", flush=True)  # 实时打印每个token，不换行，立即刷新
    elif kind == "tool_calls":  # 如果是工具调用类型
        print(f"\nTool call: {item.tool_name}({item.input})")  # 打印工具名称和输入参数
        for delta in item.output_deltas:  # 遍历工具输出的增量数据
            print(delta, end="", flush=True)  # 实时打印增量输出
        print(f"\nTool result: {item.output}")  # 打印完整的工具执行结果

final_state = stream.output  # 获取最终的代理状态输出（注：stream对象可能没有output属性）  
```

```
content: [{'type': 'text', 'text': "It's always sunny in San Francisco!"}]
step: model
```


## 自定义更新

要实时流式传输工具执行过程中的更新，可以使用`get_stream_writer`。

流式自定义更新

```python
from langchain.agents import create_agent  # 导入创建代理的函数
from langgraph.config import get_stream_writer  # 导入获取流写入器的函数（用于工具内部推送自定义数据）


def get_weather(city: str) -> str:
    """获取指定城市的天气信息"""
    writer = get_stream_writer()  # 获取流写入器实例，允许向客户端推送实时数据
    # 通过writer推送任意自定义数据（这些数据会立即被客户端接收）
    writer(f"正在查找城市数据: {city}")  # 推送进度信息：开始查询
    writer(f"已获取城市数据: {city}")   # 推送进度信息：数据获取完成
    return f"It's always sunny in {city}!"  # 返回最终天气结果


agent = create_agent(  # 创建AI代理实例
    model="claude-sonnet-4-6",  # 指定使用Claude Sonnet 4.6模型（注：此模型名称可能需要确认）
    tools=[get_weather],        # 注册工具列表
)

# 流式执行代理，stream_mode="custom"表示接收工具推送的自定义数据
for chunk in agent.stream(
    {"messages": [{"role": "user", "content": "What is the weather in SF?"}]},  # 用户输入
    stream_mode="custom",  # 自定义流模式：专门接收通过get_stream_writer推送的数据
    version="v2",          # 指定API版本（注：此参数可能已废弃）
):
    if chunk["type"] == "custom":  # 判断是否为自定义数据类型
        print(chunk["data"])       # 打印工具推送的自定义数据
```

输出

```
Looking up data for city: San Francisco
Acquired data for city: San Francisco
```

如果在工具内部添加`get_stream_writer`，则无法在 LangGraph 执行上下文之外调用该工具。

## 流式多模式

您可以通过将流模式作为列表传递来指定多个流模式：`stream_mode=["updates", "custom"]`。每个流式传输的数据块都是一个`StreamPart`字典，包含`<stream>` `type`、`ns` `<stream>` 和 `data` `<payload>` 三个键。使用`chunk["type"]` `<stream>` 可以确定流模式，`chunk["data"]`使用 `<payload>` 可以访问有效负载。

流媒体多种模式

``` python
# LangGraph 多流模式代码注释
from langchain.agents import create_agent  # 导入创建代理的函数（注：当前稳定版中可能不存在或已更名）
from langgraph.config import get_stream_writer  # 导入获取流写入器的函数，用于工具内部推送自定义数据

def get_weather(city: str) -> str:
    """获取指定城市的天气信息"""
    writer = get_stream_writer()  # 获取流写入器实例，允许向客户端推送实时数据
    writer(f"正在查找城市数据: {city}")  # 通过自定义流推送第一条进度信息
    writer(f"已获取城市数据: {city}")   # 通过自定义流推送第二条进度信息
    return f"It's always sunny in {city}!"  # 返回最终天气结果


agent = create_agent(  # 创建AI代理实例
    model="gpt-5-nano",  
    tools=[get_weather],  # 注册工具列表，包含天气查询函数
)

# 流式执行代理，同时接收节点更新和自定义数据两种流模式
for chunk in agent.stream(
    {"messages": [{"role": "user", "content": "What is the weather in SF?"}]},  # 用户输入消息
    stream_mode=["updates", "custom"],  # ✅ 多个流模式作为列表传递，同时接收两种类型
    version="v2",  # 指定API版本（注：此参数可能已废弃）
):
    # chunk 是一个 StreamPart 字典，包含三个键：
    # - "type": 流模式类型（"updates" 或 "custom"）
    # - "ns": 命名空间列表，标识数据来源节点
    # - "data": 实际的有效负载数据
    
    print(f"stream_mode: {chunk['type']}")  # 打印当前数据块的流模式类型
    print(f"content: {chunk['data']}")      # 打印数据块的实际内容
    print("\n")  # 打印空行分隔不同数据块
```

## 常见模式

以下是一些流媒体的常见应用案例示例。

### 式思维/推理令牌

[有些模型在得出最终答案之前会进行内部推理。您可以通过筛选标准内容块](https://docs.langchain.com/oss/python/langchain/messages#standard-content-blocks)来获取这些思考/推理标记`type` `"reasoning"`。模型必须启用推理输出。

要从代理流式传输思维令牌，请使用`stream_mode="messages"`并筛选推理内容块：

```python
from langchain.agents import create_agent  # 导入创建代理的函数（注：当前稳定版中可能已更名或不存在）
from langchain_anthropic import ChatAnthropic  # 导入Anthropic Claude模型聊天类
from langchain_core.runnables import Runnable  # 导入Runnable基类，用于类型提示

def get_weather(city: str) -> str:
    """获取指定城市的天气信息"""
    return f"It's always sunny in {city}!"  # 模拟天气数据，总是返回晴天
# 创建Claude模型实例

model = ChatAnthropic(
    model_name="claude-sonnet-4-6",  # 指定使用Claude Sonnet 4.6模型
    timeout=None,  # 请求超时时间，None表示不设置超时
    stop=None,     # 停止序列，None表示使用模型默认停止条件
    thinking={"type": "enabled", "budget_tokens": 5000},  # ✅ 启用Claude的思考功能，分配5000个token预算
)

# 创建AI代理实例，类型标注为Runnable（可运行对象）
agent: Runnable = create_agent(
    model=model,          # 传入Claude模型实例
    tools=[get_weather],  # 注册工具列表，包含天气查询函数
)

# 使用stream_events方法进行流式事件处理（注：此方法在当前版本中可能已废弃或API变更）
stream = agent.stream_events(
    {"messages": [{"role": "user", "content": "What is the weather in SF?"}]},  # 用户输入消息
    version="v3",  # 指定API版本（注：此参数可能已废弃）
)

# 遍历流式事件中的消息
for message in stream.messages:  # 注：stream对象可能没有messages属性
    # 遍历消息中的推理内容（Claude的思考过程）
    for token in message.reasoning:  # 注：message可能没有reasoning属性
        print(f"[thinking] {token}", end="")  # 打印思考token，不换行
    # 遍历消息中的文本内容
    for token in message.text:  # 注：message可能没有text属性
        print(token, end="", flush=True)  # 打印文本token，不换行，立即刷新
```

输出

```
[thinking] The user is asking about the weather in San Francisco. I have a tool
[thinking]  available to get this information. Let me call the get_weather tool
[thinking]  with "San Francisco" as the city parameter.
The weather in San Francisco is: It's always sunny in San Francisco!
```

无论模型提供商是谁，其工作方式都相同——LangChain 通过该属性将提供商特定的格式（Anthropic`thinking`块、OpenAI`reasoning`摘要等）规范化为标准`"reasoning"`内容块类型[`content_blocks`](https://docs.langchain.com/oss/python/langchain/messages#standard-content-blocks)。要直接从聊天模型（无需代理）流式传输推理令牌，请参阅[使用聊天模型进行流式传输](https://docs.langchain.com/oss/python/langchain/models#reasoning)。

### 流式工具调用

指定此参数`stream_mode="messages"`将流式传输代理中所有 LLM 调用生成的增量消息块。要访问已解析工具调用的完整消息：
1. [如果这些消息在状态](https://docs.langchain.com/oss/python/langchain/short-term-memory)中被跟踪（如在模型节点中[`create_agent`](https://docs.langchain.com/oss/python/langchain/agents)），则可以使用[状态更新](https://docs.langchain.com/oss/python/langchain/streaming#agent-progress)`stream_mode=["messages", "updates"]`来访问已完成的消息（如下所示）。
2. 如果状态中没有跟踪这些消息，则在流循环期间使用自定义更新或聚合块

如果您的代理包含多个 LLM，请参阅下面关于[从子代理](https://docs.langchain.com/oss/python/langchain/streaming#streaming-from-sub-agents)进行流式传输的部分。

```python
from typing import Any  # 导入Any类型，用于灵活类型标注
from langchain.agents import create_agent  # 导入创建代理的函数
from langchain.messages import AIMessage, AIMessageChunk, AnyMessage, ToolMessage  

# 导入消息类型：
# - AIMessage: 完整的AI响应消息
# - AIMessageChunk: AI响应的流式片段
# - AnyMessage: 任意消息类型的联合类型
# - ToolMessage: 工具执行结果消息

def get_weather(city: str) -> str:
    """获取指定城市的天气信息"""
    return f"It's always sunny in {city}!"  # 模拟天气数据，总是返回晴天
    
# 创建AI代理实例
agent = create_agent(
    "openai:gpt-5.5",  # ❌ 指定使用OpenAI GPT-5.5模型（注意：此模型目前不存在）
    tools=[get_weather],  # 注册工具列表，包含天气查询函数
)

def _render_message_chunk(token: AIMessageChunk) -> None:
    """
    渲染流式消息片段（处理AIMessageChunk类型的token）
    
    参数:
        token: AI响应的流式片段对象
    """
    if token.text:
        # 如果有文本内容，逐段打印，用"|"分隔
        print(token.text, end="|")
    if token.tool_call_chunks:
        # 如果有工具调用片段，打印完整的工具调用信息
        print(token.tool_call_chunks)
    # 注意：所有内容都可以通过 token.content_blocks 访问
    
def _render_completed_message(message: AnyMessage) -> None:
    """
    渲染已完成的消息（处理完整的消息对象）
    
    参数:
        message: 任意类型的完整消息对象
    """
    if isinstance(message, AIMessage) and message.tool_calls:
        # 如果是AI消息且包含工具调用，打印工具调用信息
        print(f"Tool calls: {message.tool_calls}")
    if isinstance(message, ToolMessage):
        # 如果是工具消息，打印工具响应的内容块
        print(f"Tool response: {message.content_blocks}")
# 定义用户输入消息
input_message = {"role": "user", "content": "What is the weather in Boston?"}
# 流式执行代理，同时订阅两种流模式
for chunk in agent.stream(
    {"messages": [input_message]},  # 用户输入，包装为消息列表
    stream_mode=["messages", "updates"],  # ✅ 同时接收消息流和节点更新流
    version="v2",  # 指定API版本（注：此参数可能已废弃）
):
    # chunk 是 StreamPart 字典，包含 type、ns、data 三个键
    
    if chunk["type"] == "messages":
        # 处理消息流模式的数据
        token, metadata = chunk["data"]  # 解包：token是消息片段，metadata是元数据
        if isinstance(token, AIMessageChunk):
            # 如果是AI消息片段，调用渲染函数处理
            _render_message_chunk(token)
            
    elif chunk["type"] == "updates":
        # 处理节点更新流模式的数据
        for source, update in chunk["data"].items():
            # 遍历更新字典，source是节点名称，update是节点输出
            if source in ("model", "tools"):  # 只处理来自模型或工具节点的更新
                # 获取该节点输出的最后一条消息（最新消息）
                _render_completed_message(update["messages"][-1])
```

输出

```
[{'name': None, 'args': '{"', 'id': None, 'index': 0, 'type': 'tool_call_chunk'}]
[{'name': None, 'args': 'city', 'id': None, 'index': 0, 'type': 'tool_call_chunk'}]

[{'name': None, 'args': 'Boston', 'id': None, 'index': 0, 'type': 'tool_call_chunk'}]
```

#### 访问已完成的消息

如果代理的[状态](https://docs.langchain.com/oss/python/langchain/short-term-memory)中跟踪已完成的消息，则可以按照[流式传输工具调用](https://docs.langchain.com/oss/python/langchain/streaming#streaming-tool-calls)`stream_mode=["messages", "updates"]`部分所示，在流式传输期间访问已完成的消息。

在某些情况下，已完成的消息不会反映在[状态更新](https://docs.langchain.com/oss/python/langchain/streaming#agent-progress)中。如果您有权访问代理内部机制，则可以使用[自定义更新](https://docs.langchain.com/oss/python/langchain/streaming#custom-updates)在流式传输期间访问这些消息。否则，您可以在流式传输循环中聚合消息块（见下文）。请看下面的示例，其中我们将[流写入器](https://docs.langchain.com/oss/python/langchain/streaming#custom-updates)集成到一个简化的[护盾中间件](https://docs.langchain.com/oss/python/langchain/guardrails#after-agent-guardrails)中。该中间件演示了如何调用工具来生成结构化的“安全/不安全”评估：

```
from typing import Any, Literal

from langchain.agents.middleware import after_agent, AgentState
from langgraph.runtime import Runtime
from langchain.messages import AIMessage
from langchain.chat_models import init_chat_model
from langgraph.config import get_stream_writer  
from pydantic import BaseModel


class ResponseSafety(BaseModel):
    """Evaluate a response as safe or unsafe."""
    evaluation: Literal["safe", "unsafe"]


safety_model = init_chat_model("openai:gpt-5.5")

@after_agent(can_jump_to=["end"])
def safety_guardrail(state: AgentState, runtime: Runtime) -> dict[str, Any] | None:
    """Model-based guardrail: Use an LLM to evaluate response safety."""
    stream_writer = get_stream_writer()
    # Get the model response
    if not state["messages"]:
        return None

    last_message = state["messages"][-1]
    if not isinstance(last_message, AIMessage):
        return None

    # Use another model to evaluate safety
    model_with_tools = safety_model.bind_tools([ResponseSafety], tool_choice="any")
    result = model_with_tools.invoke(
        [
            {
                "role": "system",
                "content": "Evaluate this AI response as generally safe or unsafe."
            },
            {
                "role": "user",
                "content": f"AI response: {last_message.text}"
            }
        ]
    )
    stream_writer(result)

    tool_call = result.tool_calls[0]
    if tool_call["args"]["evaluation"] == "unsafe":
        last_message.content = "I cannot provide that response. Please rephrase your request."

    return None
```

### 人机交互式流媒体

1. 我们为代理配置了人机交互中间件和检查点。
2. 我们收集`"updates"`流模式下产生的中断
3. 我们用一条命令来响应这些中断。

```python
from typing import Any  # 导入Any类型，用于灵活类型标注

from langchain.agents import create_agent  # 导入创建代理的函数
from langchain.agents.middleware import HumanInTheLoopMiddleware  # 导入人机交互中间件，用于在执行工具前请求用户确认
from langchain.messages import AIMessage, AIMessageChunk, AnyMessage, ToolMessage  
# 导入消息类型：
# - AIMessage: 完整的AI响应消息
# - AIMessageChunk: AI响应的流式片段
# - AnyMessage: 任意消息类型的联合类型
# - ToolMessage: 工具执行结果消息
from langgraph.checkpoint.memory import InMemorySaver  # 导入内存检查点保存器，用于状态持久化
from langgraph.types import Command, Interrupt  # 导入Command和Interrupt类型


def get_weather(city: str) -> str:
    """获取指定城市的天气信息"""
    return f"It's always sunny in {city}!"  # 模拟天气数据，总是返回晴天


checkpointer = InMemorySaver()  # 创建内存检查点实例，用于保存对话状态

# 创建AI代理实例，配置模型、工具、中间件和检查点
agent = create_agent(
    "openai:gpt-5.5",  # ❌ 指定使用OpenAI GPT-5.5模型（注意：此模型目前不存在）
    tools=[get_weather],  # 注册工具列表，包含天气查询函数
    middleware=[
        HumanInTheLoopMiddleware(interrupt_on={"get_weather": True}),  # ✅ 添加人机交互中间件，当调用get_weather工具时触发中断，等待用户确认
    ],
    checkpointer=checkpointer,  # 启用检查点，支持状态持久化和对话历史恢复
)


def _render_message_chunk(token: AIMessageChunk) -> None:
    """
    渲染流式消息片段（处理AIMessageChunk类型的token）
    
    参数:
        token: AI响应的流式片段对象
    """
    if token.text:
        # 如果有文本内容，逐段打印，用"|"分隔
        print(token.text, end="|")
    if token.tool_call_chunks:
        # 如果有工具调用片段，打印完整的工具调用信息
        print(token.tool_call_chunks)


def _render_completed_message(message: AnyMessage) -> None:
    """
    渲染已完成的消息（处理完整的消息对象）
    
    参数:
        message: 任意类型的完整消息对象
    """
    if isinstance(message, AIMessage) and message.tool_calls:
        # 如果是AI消息且包含工具调用，打印工具调用信息
        print(f"Tool calls: {message.tool_calls}")
    if isinstance(message, ToolMessage):
        # 如果是工具消息，打印工具响应的内容块
        print(f"Tool response: {message.content_blocks}")


def _render_interrupt(interrupt: Interrupt) -> None:
    """
    渲染中断信息（处理HumanInTheLoop触发的中断）
    
    参数:
        interrupt: 中断对象，包含需要用户响应的请求
    """
    interrupts = interrupt.value  # 获取中断的值（包含action_requests列表）
    for request in interrupts["action_requests"]:  # 遍历所有需要用户操作的动作请求
        print(request["description"])  # 打印请求描述，提示用户需要做什么


# 定义用户输入消息（同时询问两个城市的天气）
input_message = {
    "role": "user",
    "content": (
        "Can you look up the weather in Boston and San Francisco?"
    ),
}
config = {"configurable": {"thread_id": "some_id"}}  # 配置参数，设置线程ID用于状态追踪
interrupts = []  # 初始化列表，用于收集所有中断事件

# 流式执行代理，同时订阅消息流和节点更新流
for chunk in agent.stream(
    {"messages": [input_message]},  # 用户输入，包装为消息列表
    config=config,  # 传入配置（包含线程ID）
    stream_mode=["messages", "updates"],  # ✅ 同时接收消息流和节点更新流
    version="v2",  # 指定API版本（注：此参数可能已废弃）
):
    # chunk 是 StreamPart 字典，包含 type、ns、data 三个键
    
    if chunk["type"] == "messages":
        # 处理消息流模式的数据
        token, metadata = chunk["data"]  # 解包：token是消息片段，metadata是元数据
        if isinstance(token, AIMessageChunk):
            # 如果是AI消息片段，调用渲染函数处理
            _render_message_chunk(token)
            
    elif chunk["type"] == "updates":
        # 处理节点更新流模式的数据
        for source, update in chunk["data"].items():
            # 遍历更新字典，source是节点名称，update是节点输出
            if source in ("model", "tools"):
                # 处理来自模型或工具节点的更新
                # 获取该节点输出的最后一条消息（最新消息）
                _render_completed_message(update["messages"][-1])
            if source == "__interrupt__":
                # 处理中断事件（HumanInTheLoop触发）
                interrupts.extend(update)  # 将中断添加到列表中
                _render_interrupt(update[0])  # 渲染第一个中断信息
```

### 来自子代理的流式传输

当代理中存在多个 LLM 时，通常需要在生成消息时消除消息来源的歧义。为此，在创建每个代理时，需要传递一个参数。然后，在流式传输模式下，[`name`](https://reference.langchain.com/python/langchain/agents/#langchain.agents.create_agent\(name\))该名称可以通过元数据中的键访问。`lc_agent_name``"messages"`下面，我们更新[流媒体工具调用](https://docs.langchain.com/oss/python/langchain/streaming#streaming-tool-calls)示例：

1. 我们将现有工具替换为一个`call_weather_agent`内部调用代理的工具。
2. `name`我们为每个代理添加一个
3. 我们[`subgraphs=True`](https://docs.langchain.com/oss/python/langgraph/use-subgraphs#stream-subgraph-outputs)在创建流时进行指定。

## 禁用流媒体

在某些应用场景中，您可能需要禁用特定模型的单个令牌流式传输。这在以下情况下非常有用：

- 利用[多智能体](https://docs.langchain.com/oss/python/langchain/multi-agent)系统来控制哪些智能体输出其内容
- 将支持流媒体的模型与不支持流媒体的模型混合使用。
- 部署到[LangSmith 平台](https://docs.langchain.com/langsmith/observability)，并希望阻止某些模型输出流式传输到客户端

`streaming=False`在初始化模型时设置。

```
from langchain_openai import ChatOpenAI

model = ChatOpenAI(
    model="gpt-5.5",
    streaming=False
)
```

部署到 LangSmith 时，`streaming=False`请对所有不希望将输出流式传输到客户端的模型进行设置。此设置需要在部署前在图代码中进行配置。

并非所有聊天模型集成都支持此`streaming`参数。如果您的模型不支持此参数，请改用`disable_streaming=True`其他方法。所有聊天模型均可通过基类使用此参数。
